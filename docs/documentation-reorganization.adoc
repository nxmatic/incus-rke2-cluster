== Usage Instructions

=== Viewing Documentation

**In GitHub/GitLab**:
- Navigate to `docs/README.adoc` for automatic rendering
- All internal links work correctly in web interface

**In VS Code**:
- Install "AsciiDoc" extension
- Open any `.adoc` file for live preview

**Generate HTML Locally**:
@codebase
[source,bash]
----
cd docs
asciidoctor README.adoc  # Creates README.html
----

=== Development Workflow

**Adding New Documentation**:

1. Create new `.adoc` file in `docs/` directory
2. Follow existing formatting standards (use existing files as templates)
3. Add entry to `docs/README.adoc` index
4. Test rendering locally before committing

**Updating Existing Documentation**:

1. Edit the relevant `.adoc` file
2. Maintain cross-references and links
3. Update related documents if needed
4. Verify rendering and link integrity

== Documentation Generation

=== Manual Generation Options

@codebase
[source,bash]
----
# Individual files
asciidoctor docs/README.adoc

# All files with custom attributes
asciidoctor \
    --attribute source-highlighter=rouge \
    --attribute icons=font \
    --attribute sectlinks \
    docs/*.adoc

# PDF generation (requires asciidoctor-pdf)
asciidoctor-pdf docs/README.adoc
----
